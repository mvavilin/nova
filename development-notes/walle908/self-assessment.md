## 2. Описание работы над проектом

В рамках командного проекта я занималась разработкой клиентской части приложения (frontend), с фокусом на аутентификацию пользователей, формы, работу с API и Socket.IO, реализацию интерактивных страниц.

## 🔧 Технологии и инструменты

В процессе работы я использовала:

- JavaScript / TypeScript
- Tailwind CSS
- Компонентный подход (внутренний Components API)
- State API (для управления состоянием)
- REST API (JWT авторизация)
- Socket.IO (для страницы Room)
- Git (PR, code review, merge)
- Unit-тестирование (vitest)

## 3. Мои Feature Components

### 🧩 1. BaseForm

Что это:
Базовый компонент для создания всех форм в приложении (Registration, Login)

Что я сделала: Реализовала универсальную форму для авторизации
Добавила: обработку submit, сбор данных формы, управление состоянием кнопки (disabled при невалидной форме).

Почему это важно:
Используется во всех формах проекта, упрощает разработку новых форм, централизует логику работы с формами.

Дополнительно:
Покрыла тестами (6 тестов), реализовала функцию сбора данных для отправки на сервер.

### 🧩 2. InputForm

Что это:
Компонент для работы с полями ввода внутри форм, включающий в себя компонененты label, input и span для вывода ошибок.

Что я сделала:
Реализовала компонент, настроила обработку ввода, валидацию и вывод ошибок, взаимодействие с формой.

Почему это важно:
Основа всех пользовательских форм, позволяет стандартизировать ввод данных.

Дополнительно:
Покрыт unit-тестами (4 теста), реализовано корректное поведение при вводе.

### 🧩 3. Страница Room

Что это: Интерактивная страница, реализующая работу пользователей внутри игровой комнаты с использованием Socket.IO.

⚙️ Функционал

- Подключение пользователя к комнате.
- Отображение участников: по командам (красная/синяя) и без команды.
- Действия пользователя:
  - выбор роли и команды
  - выход из команды
  - выход из комнаты (переход в Lobby)
- Логика запуска игры:
  - при заполнении комнаты запускается таймер
  - по завершении — переход на страницу Game
  - при выходе игрока из команды или комнаты таймер сбрасывается и комната снова переходит в режим ожидания
- Отображение названия комнаты и количества участников.
- Всплывающие сообщения о входе/выходе игроков в комнату.
- Поддержка двух языков.

🧠 Архитектура

Проблема:

- дублирование логики
- сложное управление подписками
- риск утечек памяти

Решение:

- управление подписками вынесено на уровень страницы
- компоненты отвечают только за отображение

Результат:

- снижена связность
- упрощено управление состоянием

🚧 Сложности

- Работа с жизненным циклом событий Socket.IO (подписка, обработка, отписка).
- Синхронизация клиентской логики с backend.
- Работа в условиях параллельной разработки.
- Предотвращение утечек памяти.
