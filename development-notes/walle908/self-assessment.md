## 3. Мои Feature Components

### 🧩 1. BaseForm

Что это:
Базовый компонент для создания всех форм в приложении (Registration, Login)

Что я сделала: Реализовала универсальную форму для авторизации
Добавила: обработку submit, сбор данных формы, управление состоянием кнопки (disabled при невалидной форме)

Почему это важно:
Используется во всех формах проекта, упрощает разработку новых форм, централизует логику работы с формами

Дополнительно:
Покрыла тестами (6 тестов), реализовала функцию сбора данных для отправки на сервер

### 🧩 2. InputForm (контролируемый input-компонент)

Что это:
Компонент для работы с полями ввода внутри форм

Что я сделала:
Реализовала компонент, настроила обработку ввода, валидацию, взаимодействие с формой

Почему это важно:
Основа всех пользовательских форм, позволяет стандартизировать ввод данных

Дополнительно:
Покрыт unit-тестами (4 теста), реализовано корректное поведение при вводе

### 🧩 3. Страница Room

Что это: Интерактивная страница, реализующая работу пользователей внутри игровой комнаты с использованием WebSocket.

⚙️ Функционал

- Подключение пользователя к комнате
- Отображение участников: по командам (красная/синяя) и без команды
- Действия пользователя:
  - выбор роли и команды
  - выход из команды
  - выход из комнаты (переход в Lobby)
- Логика запуска игры:
  - при заполнении комнаты запускается таймер
  - по завершении — переход на страницу Game
  - при выходе игрока из команды или комнаты таймер сбрасывается
- Отображение названия комнаты и количества участников
- Поддержка двух языков

🔌 WebSocket

Реализовано взаимодействие с сервером через события

- обработка входящих событий (состояние комнаты, списки участников)
- создание исходящих событий (действия пользователя)

🧠 Архитектура

Проблема:

- дублирование логики
- сложное управление подписками
- риск утечек памяти

Решение:

- подписки/отписки на события и отписки вынесены на уровень страницы
- компоненты отвечают только за отображение
- реализована централизованная обработка событий

Результат:

- снижена связность
- упрощено управление состоянием

🚧 Сложности

- Работа с жизненным циклом WebSocket-событий
- Синхронизация клиентской логики с backend
- Работа в условиях параллельной разработки
- Предотвращение утечек памяти
